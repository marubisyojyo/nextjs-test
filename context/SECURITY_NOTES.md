# 管理画面まわりのセキュリティ補足

今回の修正で、管理画面は `users.is_admin = true` のユーザーだけが入れるようにしました。
既存の `role = admin` も互換のため残してあるので、どちらか片方が管理者条件になっています。

## 今回入れた対策

- `/admin` 配下でログイン中ユーザーを確認
- `users` テーブルの `is_admin` と `role` を確認
- 管理者でなければ `/admin/login` に戻す
- 管理者でないユーザーが管理者ログインを通っても強制サインアウト

## 追加でやった方がいいこと

### 1. 本番では `users` テーブルの RLS を有効にする
今は開発しやすいように RLS を広めにしている可能性があります。
本番では、少なくとも一般ユーザーが他人の `is_admin` や `points_balance` を更新できないようにしてください。

### 2. admin 用の更新処理は将来的に API Route 経由にする
今の管理画面はブラウザから直接 Supabase を触っています。
本番でより堅くするなら、

- 商品登録
- 商品編集
- 商品削除
- ポイント付与

のような重要操作は `/api/admin/...` に寄せて、サーバー側で `is_admin` を再確認してから処理するのが安全です。

### 3. `likes` など開発中に RLS を切ったテーブルは本番前に見直す
今回いいね動作のために RLS を止めたテーブルがある場合、本番前に必ず見直してください。

### 4. `.env.local` は絶対に配布・公開しない
今回のZIPにも入れないのが正解です。
GitHubに上げるときも `.env.local` が入らないようにしてください。

## 運用メモ

- 自分の管理者アカウントだけ `is_admin = true`
- 一般会員は必ず `false`
- 新規会員のデフォルト値は `false`
